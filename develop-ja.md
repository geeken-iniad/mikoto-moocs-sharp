# 開発ガイド

このガイドでは、開発環境のセットアップとプロジェクトのビルド方法について説明します。

## プロジェクト構成

このリポジトリは、主に2つのプロジェクトを含むモノレポ構成になっています。

- **`./` (ルート):** `wxt`でビルドされたブラウザ拡張機能。
- **`userscript/`:** `Vite`と`React`でビルドされたユーザースクリプトプロジェクト。

これら2つのプロジェクトは連携して動作するように設計されています。まず`userscript`プロジェクトがビルドされ、その出力として単一のJavaScriptファイル`mikoto-userscript.js`が生成されます。このファイルは、ルートプロジェクトの`public/`ディレクトリに配置されます。

その後、ブラウザ拡張機能はバックグラウンドスクリプト(`entrypoints/background.ts`)を使用して、`mikoto-userscript.js`を対象のWebページ(`https://moocs.iniad.org/*`)にプログラム的に注入します。

## 開発フロー

1.  **依存関係のインストール:**
    ルートディレクトリで以下のコマンドを実行して、拡張機能とユーザースクリプトプロジェクトの両方の依存関係をインストールします:
    ```sh
    pnpm i
    ```
    ルートの`package.json`にある`postinstall`スクリプトが、`userscript/`ディレクトリで自動的に`pnpm install`を実行します。

2.  **開発サーバーの実行:**
    ブラウザ拡張機能の開発サーバーを起動するには、以下を実行します:
    ```sh
    pnpm dev
    ```
    これにより、拡張機能のコードの変更が監視されます。ただし、これはユーザースクリプトを自動的に再ビルド**しません**。`userscript/`プロジェクトに変更を加えた場合は、手動で再ビルドする必要があります。

## ビルドプロセス

プロジェクトをビルドするには、ルートディレクトリから以下の`pnpm`スクリプトを使用できます:

-   **`pnpm build:userscript`**: ユーザースクリプトのみをビルドします。ユーザースクリプトのみに変更を加えている場合に便利です。
-   **`pnpm build`**: ユーザースクリプトとブラウザ拡張機能の両方をビルドします。まず`build:userscript`を実行し、次に拡張機能をビルドします。
-   **`pnpm build:firefox`**: Firefox用にユーザースクリプトとブラウザ拡張機能をビルドします。

## 配布

配布用のzipファイルを作成するには、次のコマンドを使用します:

-   **`pnpm zip`**: Chrome用のzipファイルを作成します。
-   **`pnpm zip:firefox`**: Firefox用のzipファイルを作成します。
